InstallCài đặt

Linux / macOS / Windows · Python ≥3.9 · git · Claude Code

Installing is one command — you paste it, and nothing else. It downloads the latest release, verifies the checksum, extracts it, checks dependencies, installs into your repo, and verifies the wiring (--strict) — in that order, fail-fast at the first problem. No preflight to run first, no follow-up scripts to chain. Requirements: Python ≥ 3.9, git, and Claude Code.Cài đặt là một lệnh — bạn dán vào, hết. Nó tải bản release mới nhất, kiểm checksum, giải nén, kiểm phụ thuộc, cài vào repo, rồi xác nhận wiring (--strict) — theo đúng thứ tự đó, dừng ngay ở lỗi đầu tiên. Không phải chạy preflight trước, không script nối tiếp. Yêu cầu: Python ≥ 3.9, git, và Claude Code.

Install online — one lineCài online — một dòng

Run this from inside the repo you want the harness in. It resolves the latest release from GitHub, downloads the bundle, and installs it — you never touch a version number or a download link.Chạy ngay trong repo bạn muốn có harness. Nó tự tìm bản mới nhất trên GitHub, tải bundle, và cài — bạn không cần đụng tới số phiên bản hay đường tải nào.

Linux / macOSLinux / macOS

curl -fsSL https://hieubui2409.github.io/sdlc-harness-release/install.sh | sh

Windows (PowerShell)Windows (PowerShell)

irm https://hieubui2409.github.io/sdlc-harness-release/install.ps1 | iex
🎛️
Pick a version or preview first. The same one line takes flags: --version=5.4.0 pins a release, -i / --interactive lists every release and previews its changelog before you confirm, and -n / --dry-run prints exactly what would be downloaded (URL + expected sha256) and installs nothing. To pass a flag through the pipe: curl -fsSL <url> | sh -s -- --interactive.Chọn phiên bản hoặc xem trước. Cùng một dòng nhận cờ: --version=5.4.0 ghim một bản, -i / --interactive liệt kê mọi bản và xem changelog trước khi bạn xác nhận, còn -n / --dry-run in đúng thứ sẽ tải (URL + sha256 kỳ vọng) và không cài gì. Muốn truyền cờ qua pipe: curl -fsSL <url> | sh -s -- --interactive.

Watch one thing: the closing verify. After it copies harness/ and wires the hooks into .claude/settings.json (additively — your own hooks survive), the installer runs verify_install --strict as the final gate. A clean run ends with zero drift; that output is your proof the install is sound. real outputChỉ cần nhìn một thứ: verify chốt. Sau khi copy harness/ và gắn hook vào .claude/settings.json (cộng thêm — hook của bạn vẫn còn), trình cài chạy verify_install --strict làm gate cuối. Một lần chạy sạch kết thúc với zero drift; đầu ra đó là bằng chứng bản cài đúng. đầu ra thật

curl -fsSL …/install.sh | sh
verify_install OK: manifest + hook registration consistent
🐍
Python is required on the target — not just to install. Every hook runs as a python3 script on each tool call, so the machine that runs the harness needs Python ≥ 3.9. The installer fails clearly up front if it is missing. No Python? Install it first (apt install python3 · brew install python · python.org), then re-run. We deliberately do not bundle a venv: a venv references a base interpreter (it does not contain Python) and is not portable across machines — it would not help the no-Python case and would break on a different OS. The whole dependency surface is three pip packages (PyYAML + pytest + defusedxml), which the installer's deps check installs for you.Máy đích bắt buộc có Python — không chỉ để cài. Mỗi hook chạy như script python3 ở từng lần gọi tool, nên máy chạy harness cần Python ≥ 3.9. Trình cài báo rõ ngay từ đầu nếu thiếu. Chưa có Python? Cài trước (apt install python3 · brew install python · python.org) rồi chạy lại. Chúng tôi cố ý không đóng gói venv: venv tham chiếu tới một interpreter gốc (nó không chứa Python) và không di động giữa các máy — gói venv không cứu được trường hợp thiếu Python mà còn vỡ trên OS khác. Toàn bộ phụ thuộc chỉ là ba gói pip (PyYAML + pytest + defusedxml), và bước kiểm deps của trình cài lo việc cài.

Offline — install a downloaded bundleOffline — cài từ bundle đã tải

No network on the target? Download the release bundle harness-<version>.tar.gz (with its .sha256) and the standalone install.sh elsewhere, copy them over, and point the installer at the bundle and your repo. Same five steps, same closing verify — it just skips the download.Máy đích không có mạng? Tải bundle release harness-<version>.tar.gz (cùng .sha256) và install.sh độc lập ở nơi khác, copy sang, rồi trỏ trình cài vào bundle và repo của bạn. Cùng năm bước, cùng verify chốt — chỉ bỏ bước tải.

sh install.sh harness-<version>.tar.gz ../my-repo

🧭
Personal-first — local advises, the cage never loosens. The workflow gates at push / pr / ship are advisory: they warn and trace, they do not block you, so you iterate freely. What stays hard on every machine, with no CI required: the agent RBAC cage, the safety floors (destructive-to-protected + secret), and the artifact-forgery block. Want a hard remote gate too? The shipped .github/workflows/receipts-gate.yml is optional — wire it into your own CI to require plan receipts before merge. Nothing about the harness assumes a remote server enforcing you.Personal-first — local khuyên, cũi không bao giờ nới. Các gate quy trình ở push / pr / ship là advisory: chúng cảnh báo và ghi trace, không chặn bạn, nên bạn lặp thoải mái. Cái luôn cứng ở mọi máy, không cần CI: cũi RBAC agent, các safety floor (destructive-to-protected + secret), và chặn giả-mạo-artifact. Muốn thêm một gate remote cứng? File .github/workflows/receipts-gate.yml đi kèm là tùy chọn — tự wire vào CI của bạn để đòi plan receipt trước khi merge. Harness không giả định có server từ xa nào ép bạn.

From a clone (advanced)Từ một clone (nâng cao)

If you already have the harness checked out, skip the bundle and run the installer directly — same writes, same final verify. Every mutation is idempotent and previewable with --dry-run. Keep the extracted harness outside the repo you install into: if source == target the copy is a no-op and nothing installs.Nếu đã có harness sẵn, bỏ qua bundle và chạy thẳng trình cài — cùng cách ghi, cùng verify cuối. Mọi thay đổi đều idempotent và xem-trước được bằng --dry-run. Để thư mục harness vừa giải nén ở ngoài repo định cài: nếu source trùng target thì bước copy thành no-op, không cài được gì.

python3 harness/install/install.py --target ../my-repo

Direct-installer flags: --non-interactive / --yes (never prompt — the automation path), --dry-run (plan every write, change nothing), --no-track (run without committing harness/ into your product git), --uninstall (reverse the settings + pre-push edits; harness/ is left in place).Cờ của trình cài trực tiếp: --non-interactive / --yes (không hỏi — đường tự động), --dry-run (lập kế hoạch mọi lần ghi, không đổi gì), --no-track (chạy mà không commit harness/ vào git sản phẩm), --uninstall (đảo ngược chỉnh settings + pre-push; harness/ vẫn để nguyên).

📐
The installer never fabricates your standards — it warns when harness/standards/ is missing or thin. Run scaffold_standards.py --type system-architecture|code-standards for a TBD skeleton to fill. Once installed, tune behavior on the Configure page; every knob (file, default, how to change it, env override) is mapped in harness/rules/config-reference.md.Trình cài không bao giờ bịa standards của bạn — nó cảnh báo khi harness/standards/ trống hoặc sơ sài. Chạy scaffold_standards.py --type system-architecture|code-standards để có khung TBD mà điền. Cài xong, chỉnh hành vi ở trang Cấu hình; mọi núm (file, mặc định, đổi cách nào, env override) liệt kê trong harness/rules/config-reference.md.
✅
After installing: enable the plugin with /reload-plugins in Claude Code (or restart it), then run your first loop in the Quick start. Re-verify any time with python3 harness/scripts/verify_install.py --strict.Sau khi cài: bật plugin bằng /reload-plugins trong Claude Code (hoặc khởi động lại), rồi chạy vòng đầu tiên trong Bắt đầu. Verify lại bất cứ lúc nào với python3 harness/scripts/verify_install.py --strict.

FAQHỏi đáp

Is curl … | sh safe?curl … | sh có an toàn không?
The script is served over HTTPS from the release site, and it verifies the bundle's sha256 sidecar before trusting a byte of it — plus a tar-escape guard that refuses any absolute, .., or out-of-tree member. Prefer to read first? Open the URL in a browser, or run with --dry-run to see the exact download plan before anything installs. For a fully offline path, download the bundle and run the local install.sh instead.Script được phục vụ qua HTTPS từ trang release, và nó kiểm sha256 sidecar của bundle trước khi tin một byte nào — kèm một tar-escape guard từ chối mọi member tuyệt đối, .., hay ra-ngoài-cây. Muốn đọc trước? Mở URL trên trình duyệt, hoặc chạy với --dry-run để xem đúng kế hoạch tải trước khi cài gì. Muốn hoàn toàn offline, tải bundle và chạy install.sh cục bộ.
What if the target machine has no Python?Nếu máy đích không có Python thì sao?
Then the harness cannot run at all — every hook is a python3 script, so Python ≥ 3.9 is a runtime requirement, not just an install-time one. The installer detects this and stops with the install command for your OS. Install Python first, then re-run.Thì harness không chạy được — mọi hook là script python3, nên Python ≥ 3.9 là yêu cầu lúc chạy, không chỉ lúc cài. Trình cài phát hiện và dừng kèm lệnh cài cho OS của bạn. Cài Python trước, rồi chạy lại.
Do the post-install tests run automatically?Bộ test sau cài có tự chạy không?
No — they are off by default as of 5.4.0. One command installs and runs the strict integrity verify, and that is enough to confirm the install. Add --run-tests (-RunTests on PowerShell) if you want to also watch the suite go green in your environment.Không — từ 5.4.0 chúng mặc định tắt. Một lệnh cài và chạy verify integrity nghiêm ngặt, thế là đủ xác nhận bản cài. Thêm --run-tests (-RunTests trên PowerShell) nếu muốn xem thêm bộ test chạy xanh trong môi trường của bạn.
Will installing touch files I didn't expect?Cài đặt có đụng file tôi không ngờ không?
Run the direct installer with --dry-run first — it plans every write and changes nothing. The real run is idempotent, merges .claude hooks additively (your own survive), and --uninstall reverses the settings + pre-push edits (it leaves harness/ in place; delete that folder to remove it fully).Chạy trình cài trực tiếp với --dry-run trước — nó lập kế hoạch mọi lần ghi và không đổi gì. Lần chạy thật là idempotent, gộp hook .claude theo kiểu cộng thêm (hook của bạn vẫn còn), và --uninstall đảo ngược chỉnh settings + pre-push (để harness/ nguyên; xóa thư mục đó để gỡ hẳn).
