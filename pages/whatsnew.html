What's newCó gì mới

v5.4.0 · 2026-07-24

Install-and-onboarding release. The installer became a self-downloading one-command bootstrap, and — the part the tour was missing until now — a coverage-grid planning mode for hs:plan / hs:cook, six new methodology & governance rules with a self-review nudge, and methodology references wired across existing skills. This page is the fast tour of all of it; the deep dive is on Methodology.Bản release về cài đặt & onboarding. Trình cài trở thành một lệnh tự-tải bootstrap, và — phần mà tour còn thiếu tới giờ — một chế độ coverage-grid cho hs:plan / hs:cook, sáu rule phương pháp & quản trị mới kèm nudge tự-review, và các tham chiếu phương pháp gắn xuyên các skill có sẵn. Trang này là tour nhanh cho tất cả; đi sâu ở Phương pháp.

🟦 Coverage-grid planning mode🟦 Chế độ coverage-grid

A redesigned coverage engine for hs:plan --grid / hs:cook: it splits coverage into breadth and depth, builds a covering-array skeleton from the plan's own features/risks, pre-registers constraint rules anchored to the approved plan hash, and credits N/A cells only against that plan. A hard presence gate — the verdict stays advisory.Một engine coverage thiết kế lại cho hs:plan --grid / hs:cook: tách coverage thành bề rộng và chiều sâu, dựng skeleton covering-array từ chính features/risks của plan, pre-register các rule ràng buộc neo vào hash plan đã duyệt, và chỉ credit ô N/A theo plan đó. Một presence gate cứng — verdict vẫn advisory.

📐 Six methodology + governance rules📐 Sáu rule phương pháp + quản trị

adr-trigger, anti-tautology-detection, claim-verification, methodology-review, persistent-risk-register, regression-sentinel — wired with a self-governance review-nudge hook, an import-graph tautology detector, a universal-claim verification engine, a never-delete regression sentinel, and a project-wide risk register. Rule layer 22 → 28.adr-trigger, anti-tautology-detection, claim-verification, methodology-review, persistent-risk-register, regression-sentinel — kèm hook nudge tự-review, bộ phát hiện tautology theo import-graph, engine verify claim phổ quát, sentinel regression không-xoá, và một risk register toàn dự án. Lớp luật 22 → 28.

⚡ Self-downloading one-command install⚡ Cài một lệnh tự-tải

curl … | sh / irm … | iex now bootstrap over the network — no bundle fetches the LATEST release, a URL fetches a specific one, a local tarball installs fully offline. Adds -i/--interactive, -n/--dry-run (print the fetch plan + expected sha256), and a --version=X pin. Also changed & fixedCòn đổi & sửa

ChangedĐã đổi

  • Post-install test suite is opt-in; the strict integrity check still confirms every install.Bộ test sau cài là tùy chọn; kiểm strict vẫn xác nhận mọi bản cài.
  • CI modernized — Actions on Node 24, install-matrix probes every dispatch mode.CI hiện đại hoá — Actions trên Node 24, install-matrix thử mọi chế độ dispatch.
  • The release bundle no longer ships the dev/CI test tree.Bundle release không còn kèm cây test dev/CI.

FixedĐã sửa

  • Grid: fail closed when pre-registration can't anchor the ruleset hash; reject forged rule refs; close N/A denominator gaming.Grid: fail-closed khi pre-register không neo được hash ruleset; từ chối rule ref giả; chặn gaming mẫu số N/A.
  • Ignore a foreign payload cwd in the agent isolation floor under a global install.Bỏ qua cwd payload lạ ở isolation floor dưới global install.
  • Windows install path hardening (UTF-8 stdout, deps into py -3, -SkipTests).Gia cố đường cài Windows (UTF-8 stdout, deps vào py -3, -SkipTests).
